**PRODUCT REQUIREMENTS:**
-   **Core Platform**: Landing page, Admin login, and User login.
-   **Admin Panel**:
    -   Manage project settings and logo.
    -   Dynamically add and configure provider APIs (Mobile, DTH, BillPayment).
    -   Define commission structures (Percentage/Flat) based on user profiles and operators.
    -   Create an intelligent routing engine (user, amount, operator, profile, state, API balance, success ratio).
    -   View API balance and set low balance alerts (Email, SMS, Webhook, Telegram).
    -   Manage users, including editing profiles and manual wallet adjustments.
    -   View comprehensive reports.
-   **User Management**:
    -   Registration (Mobile + OTP), Login (JWT).
    -   User roles: Retailer, Distributor, Super Distributor, API User.
    -   KYC Management (PAN, Aadhaar, GST).
-   **Wallet System (CRITICAL)**:
    -   Must use a double-entry accounting ledger system.
    -   Every transaction must be atomic (debit recharge amount, credit commission separately).
    -   Manual wallet credit/debit for users by admin.
-   **Recharge Flow**:
    -   Secure transaction flow with wallet locking, pending status, and success/failure handling.
    -   Automatic retry/failover to a second API if the first one fails in real-time.
    -   Manual retry option for failed recharges.
    -   Transaction status check mechanism.
-   **API & Security**:
    -   Provide a customer-facing API for resellers (Recharge, Status check) with API key and IP restriction.
    -   Implement a strong, configurable response code mapping system.
-   **Reporting**:
    -   User ledger, operator-wise, date-wise, commission, API profit, state-wise, and success/fail ratio reports.
-   **Scalability**:
    -   The architecture should handle heavy loads using message queues, database transactions, and row-level locking.

**User's preferred language**: English

**what currently exists?**
A functional full-stack application with a NestJS backend and a React frontend. The backend environment has been fixed and is now stable. The application supports user/admin roles, JWT authentication, and a double-entry wallet ledger system.

The Admin Panel has been significantly enhanced with functional pages for:
